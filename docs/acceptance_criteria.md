# Критерии приемки проекта (практика)

Документ фиксирует критерии приемки учебного Django-проекта IBSec LMS для практики.

## 1. Базовое соответствие стеку

1. Проект реализован на Django.
- Статус: соответствует (`Django==4.2.13` в `requirements.txt`).

2. Используется SQL-БД.
- Статус: соответствует (SQLite по умолчанию, PostgreSQL поддерживается настройкой).

3. Проект ведется в Git.
- Статус: соответствует (репозиторий с `.git`).

## 2. Функциональные требования практики

1. Наличие 3 ролей пользователей.
- Требование: минимум три роли.
- Статус: соответствует (`employee`, `instructor`, `admin`).

2. Личный кабинет пользователя.
- Требование: отдельная страница/функция личного кабинета.
- Статус: соответствует (`/accounts/profile/`, `/reports/my/`).

3. Админ-панель.
- Требование: административный контур управления.
- Статус: соответствует (`/admin/`).

4. Отчеты.
- Требование: формирование и просмотр отчетных данных.
- Статус: соответствует (`/reports/analytics/`, `/reports/export/`).

5. Не менее 10 страниц интерфейса.
- Требование: 10+ маршрутов/страниц.
- Статус: соответствует (в проекте есть 10+ пользовательских и служебных страниц).

## 3. Развертывание и хостинг

1. Подготовленность к разворачиванию.
- Статус: соответствует на уровне учебного проекта (настройки и зависимости присутствуют).

2. Хостинг (требование практики).
- Статус: требует подтверждения факта публикации в выбранной среде (например, VPS/PaaS).
- Комментарий: кодовая база готова к развертыванию; факт публичного размещения подтверждается отдельно в рамках сдачи практики.

## 4. Приемочные проверки

1. Приложение запускается локально по инструкции из README.
2. Пользователь может зарегистрироваться, войти и открыть профиль.
3. Преподаватель/администратор может создать курс и добавить урок.
4. Сотрудник может пройти тест и получить результат.
5. Отчетность и CSV-выгрузка доступны роли `instructor`/`admin`.
6. Основные маршруты открываются без ошибок при корректных правах доступа.

## 5. Результат приемки

Проект считается принятым для практики при выполнении пунктов разделов 1-2 и успешном прохождении проверок из раздела 4. Пункт по хостингу фиксируется отдельным подтверждением публикации.
