# IBSec LMS

**IBSec LMS** — учебно-корпоративная система для обучения сотрудников основам информационной безопасности и проверки знаний.
Проект реализован на Django и предназначен для демонстрации типового LMS-сценария: изучение курсов, прохождение тестов, просмотр результатов и аналитики.

Репозиторий подходит для:
- локального запуска без сложной настройки;
- демонстрации курсовой / дипломной работы;
- подготовки скриншотов интерфейса;
- расширения под собственные сценарии корпоративного обучения.

---

## О проекте

Система моделирует внутреннюю платформу обучения сотрудников по тематике информационной безопасности.
В приложении реализованы роли пользователей, каталог курсов, уроки разных типов, тестирование, личный прогресс и базовая аналитика для преподавателя или администратора.

Проект построен как монолитное Django-приложение и разбит на отдельные приложения по зонам ответственности:
- `accounts` — авторизация, регистрация, профиль и роли пользователей;
- `courses` — курсы и уроки;
- `quizzes` — тесты, вопросы, варианты ответов и результаты;
- `reports` — отчёты, аналитика и экспорт CSV;
- `notifications` — модель уведомлений и отправка email-уведомлений;
- `core` — главная страница и служебная команда заполнения демо-данных.

## Функциональность

В проекте реализованы следующие возможности.

### Пользователи и роли
- вход и выход из системы;
- регистрация нового пользователя;
- профиль пользователя;
- роли:
  - **Администратор**;
  - **Преподаватель**;
  - **Сотрудник**.

### Работа с курсами
- просмотр списка опубликованных курсов;
- просмотр страницы курса;
- создание курсов преподавателем или администратором;
- добавление уроков в курс.

### Уроки
Поддерживаются три типа учебных материалов:
- текст;
- ссылка;
- файл.

### Тестирование
- тест по курсу;
- вопросы с одним вариантом ответа;
- автоматическая проверка результата;
- отображение итогового балла в процентах;
- определение факта успешного прохождения по пороговому баллу.

### Отчёты и аналитика
- личный прогресс пользователя;
- просмотр истории прохождений тестов;
- аналитика по тестам для преподавателя и администратора;
- экспорт результатов в CSV.

### Дополнительно
- консольная отправка email-уведомлений;
- модель уведомлений;
- готовая management-команда `seed_demo` для заполнения системы демо-данными.

## Быстрый старт

Ниже приведён базовый сценарий локального запуска.

### 1. Создайте и активируйте виртуальное окружение

```bash
python -m venv .venv
```

Для Linux / macOS:

```bash
source .venv/bin/activate
```

Для Windows:

```bash
.venv\Scripts\activate
```

### 2. Установите зависимости

```bash
pip install -r requirements.txt
```

### 3. Примените миграции

```bash
python manage.py migrate
```

### 4. Запустите демо-наполнение базы данных

```bash
python manage.py seed_demo
```

Команда создаст демонстрационных пользователей, курс, уроки и тест.

### 5. При необходимости создайте суперпользователя

```bash
python manage.py createsuperuser
```

### 6. Запустите сервер разработки

```bash
python manage.py runserver
```

После запуска приложение будет доступно по адресу:

```text
http://127.0.0.1:8000/
```

## Маршруты приложения

Основные URL проекта:

- `/` — главная страница;
- `/accounts/login/` — вход;
- `/accounts/signup/` — регистрация;
- `/accounts/profile/` — профиль пользователя;
- `/courses/` — список курсов;
- `/courses/<id>/` — страница курса;
- `/quizzes/course/<course_id>/` — вход в тест по курсу;
- `/quizzes/take/<quiz_id>/` — прохождение теста;
- `/reports/my/` — личный прогресс;
- `/reports/analytics/` — аналитика;
- `/reports/export/` — экспорт результатов в CSV.

## Ролевая модель доступа

В проекте предусмотрено разграничение прав:

- **Сотрудник**
  - просматривает опубликованные курсы;
  - проходит тестирование;
  - видит свой прогресс.

- **Преподаватель**
  - создаёт курсы;
  - добавляет уроки;
  - просматривает аналитику;
  - экспортирует результаты.

- **Администратор**
  - обладает возможностями преподавателя;
  - может дополнительно управлять системой через административную панель Django.
